# Разбитые сердца

Мини-игра для Telegram Mini App в жанре аркады-ловли: **Катя ловит сердца, которые бросает Тарас**.

## О игре (описание)

**Разбитые сердца** — короткая драматичная аркада о паре, которой не суждено быть вместе в реальном мире. Тарас и Катя существуют в одном пространстве только здесь: он сидит на люстре и кидает вниз свои сердца; она бегает по полу и либо ловит их — и тогда в финале они оказываются вместе, над ними сердечки и они берутся за руки, — либо разбивает. Каждое упавшее сердце бьётся о пол и разбивается; жизни в игре — это тоже сердца Тараса: при пяти промахах он падает с люстры. Победа — поймать все 30 сердец и сохранить хотя бы одну жизнь: Тарас спрыгивает вниз, они встречаются. Поражение — пять промахов: все сердца разбиты, Тарас падает. Игра строится на контрасте «поймать = шанс быть вместе» и «промах = разбитое сердце», без морализаторства, с чёткой механикой и быстрым раундом.

**Жанр:** аркада, ловля предметов, одна сессия 1–2 минуты. **Платформа:** Telegram Mini App. **Аудитория:** игроки, которым нравятся простой геймплей и короткая эмоциональная история в одном раунде.

## Механика (кратко)

Пол внизу экрана; Катя на полу, Тарас на люстре сверху. 30 сердец за раунд, 5 жизней. Респаун случайный (300–1250 мс), 5% шанс «два подряд»; скорость падения у каждого сердца случайная (1.1–1.5×). Пауза с обратным отсчётом 3-2-1 по 0,5 с. Управление: тап левая/правая половина экрана (или стрелки на десктопе). Единый стиль UI: подложки, тёмная типографика в меню, паузе, «Об игре», победе и проигрыше.

## Запуск локально

```bash
cd broken-hearts
npm install
npm run dev
```

Открыть в браузере указанный адрес (например `http://localhost:5173/broken-hearts/`). Игра отображается вертикально (9:16), по бокам — градиентный фон (letterbox).

## Сборка и деплой

```bash
npm run build
```

### Почему не запускается по ссылке GitHub Pages?

Игра — это **собранный** фронтенд (Vite build). Если в настройках репозитория **Pages → Source** указана ветка **main**, отдаётся исходный код: браузер запрашивает `/src/main.js` и другие модули, которых на сайте нет в нужном виде, поэтому игра не грузится.

**Что сделать:**

1. **Собрать и выложить билд в ветку `gh-pages`:**
   ```bash
   npm install
   npm run deploy
   ```
   (скрипт `deploy` собирает проект и пушит папку `dist` в ветку `gh-pages`.)

2. **В GitHub:** репозиторий → **Settings → Pages** → **Source:** Deploy from a branch → **Branch:** `gh-pages`, папка **/ (root)** → Save.

3. **Открывать игру по полному URL:**  
   `https://bogdanuz.github.io/broken-hearts/`  
   (обязательно путь `/broken-hearts/`, не корень домена.)

**Важно: ветки `main` и `gh-pages` — разные по смыслу.** Не создавать pull request из `gh-pages` в `main` и не мержить их. В `main` — исходный код (src/, public/, package.json); в `gh-pages` — только результат сборки (содержимое dist/), его заполняет команда `npm run deploy`. GitHub после пуша в `gh-pages` может показывать баннер «Compare & pull request» — его можно игнорировать. Обновление сайта: правки в коде → коммит в `main` → снова `npm run deploy`.

После деплоя подключить Mini App в BotFather: URL = `https://bogdanuz.github.io/broken-hearts/`.

## Адаптивность и Telegram

- **Телефоны:** игра рассчитана на открытие из приложения Telegram с любого телефона (Mini App). Разрешение 480×854, масштаб FIT, центрирование; управление тапом по половинкам экрана. Подробности — **SPEC.md**, раздел «Адаптивность на телефонах и в Telegram».
- **Без сервера:** игра — статичный фронтенд на GitHub Pages; своего бэкенда нет. Mini App подключается в BotFather указанием URL игры; токен бота **не используется в коде и не хранится в репозитории**. Если позже понадобится серверная логика (например ответ бота на /start) — только через Supabase; токен хранить **только в Supabase Vault**, не в коде. Подробности — **SPEC.md**, раздел «Секреты и бот (Supabase)».

**Кнопка «Поделиться»:** в Telegram (Mini App) открывает стандартное окно «Поделиться» через `t.me/share/url`. В браузере на телефоне — нативный шаринг (если есть), иначе копирование ссылки в буфер с подсказкой «Ссылка скопирована». Кнопка всегда кликабельна; эффект зависит от окружения.

**Музыка:** при входе в меню воспроизведение запускается, если включено в настройках. Если браузер блокирует автозапуск до первого тапа — музыка стартует при первом нажатии на экран. Переключатель «Музыка вкл/выкл» сразу останавливает или снова запускает трек меню.

*Инструкции по деплою и тексты для бота хранятся локально в DEPLOY.md и BOT_CONTENT.md (эти файлы не коммитятся в репозиторий).*
